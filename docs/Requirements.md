# 闲鱼管理系统 - 需求文档

## 1. 项目概述

开发一套闲鱼管理系统，用于管理闲鱼平台的账号、商品、订单等业务，支持自动化运营功能。

## 2. 技术栈

| 层级 | 技术选型 |
|------|---------|
| 后端 | PHP 8.2 + MySQL 8.0 |
| 前端 | React 18 + Ant Design 5 + Vite |
| 容器 | Docker + Docker Compose |
| 服务器 | Nginx + PHP-FPM |

## 3. 功能模块

### 3.1 账号管理
- 闲鱼账号的增删改查
- 账号状态监控（在线/离线/异常）
- 账号分组管理
- 批量导入/导出

### 3.2 商品管理
- 商品信息维护（标题、描述、价格、图片）
- 商品上下架控制
- 商品分类管理
- 库存管理

### 3.3 订单管理
- 订单列表查询
- 订单状态跟踪
- 订单导出
- 退款处理

### 3.4 自动回复
- 关键词触发自动回复
- 回复模板管理
- 回复延迟设置
- 开启/关闭控制

### 3.5 指定回复
- 针对特定用户/商品的定制回复
- 优先级设置
- 时间段配置

### 3.6 卡券管理
- 卡券库存管理
- 卡券分类
- 卡券导入/导出
- 使用状态追踪

### 3.7 自动发货
- 订单支付后自动发送卡券
- 发货规则配置
- 发货日志

### 3.8 消息提醒
- 新订单提醒
- 库存预警
- 异常告警
- 浏览器通知 + 声音提醒

### 3.9 在线聊天
- 实时聊天界面
- 聊天记录查询
- 快捷回复
- 多会话管理

## 4. API接口规范

### 4.1 认证方式
- API Key + Secret 签名认证
- 支持易语言等外部程序接入

### 4.2 接口格式
- RESTful 风格
- JSON 数据格式
- 统一响应结构

### 4.3 核心接口
- `/api/v1/auth/*` - 认证相关
- `/api/v1/accounts/*` - 账号管理
- `/api/v1/products/*` - 商品管理
- `/api/v1/orders/*` - 订单管理
- `/api/v1/coupons/*` - 卡券管理
- `/api/v1/messages/*` - 消息相关
- `/api/v1/replies/*` - 回复管理

## 5. UI要求

- 现代化设计风格
- 响应式布局（PC/平板/手机）
- 深色/浅色主题切换
- 流畅的交互动画

## 6. Docker交付标准

- 支持 ARM64 + AMD64 架构
- `docker compose up --build -d` 一键启动
- 前端端口: 8084（Nginx 代理）
- 后端端口: 8083（Apache + PHP）
- 数据库端口: 3309
