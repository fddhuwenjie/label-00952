# 闲鱼管理系统

一套功能完整的闲鱼店铺管理系统，支持多账号管理、商品管理、订单管理、自动回复、卡券发货等功能。

## How to Run

```bash
# 克隆项目后，在根目录执行
docker-compose up --build -d

# 等待所有服务启动完成（约1-2分钟）
docker-compose ps

# 查看日志
docker-compose logs -f
```

启动成功后访问：**http://localhost:8084**

| 服务 | 端口 | 说明 |
|------|------|------|
| frontend-admin | 8084 | 管理后台前端 |
| backend | 8083 | PHP API 后端 |
| mysql | 3309 | MySQL 数据库 |

**API 接口认证（用于易语言接入）：**
- API Key: 登录系统后在【系统设置】→【API接口】中查看
- API Secret: 系统自动生成，可点击刷新重新生成

> 注意：前端通过 Nginx 代理访问后端 API，无需直接访问 8083 端口

## 测试流程

系统已预置测试数据，登录后可按以下流程进行功能测试：

### 1. 账号管理测试

1. 登录系统，点击左侧菜单【账号管理】
2. 查看预置的4个测试账号（数码主号、数码副号、服装主号、百货测试号）
3. 点击【分组管理】Tab，查看预置的3个分组（数码店铺、服装店铺、日用百货）
4. 测试添加分组：点击【添加分组】，输入名称如"测试分组"
5. 测试添加账号：点击【添加账号】，选择刚创建的分组
6. 测试编辑/删除功能

### 2. 商品管理测试

1. 点击左侧菜单【商品管理】
2. 查看预置的5个测试商品
3. 测试添加商品：点击【添加商品】，填写标题、价格、库存等信息
4. 测试商品状态切换：草稿→上架→下架
5. 测试商品筛选：按状态、分类筛选

### 3. 订单管理测试

1. 点击左侧菜单【订单管理】
2. 查看预置的5个测试订单（不同状态）
3. 测试发货：找到"已支付"状态的订单，点击【发货】
4. 测试订单状态流转：待支付→已支付→已发货→已完成
5. 测试订单筛选：按状态、日期筛选

### 4. 卡券管理测试

1. 点击左侧菜单【卡券管理】
2. 查看预置的7个测试卡券
3. 测试批量导入：点击【批量导入】，输入多行卡密
4. 关联商品测试：将卡券与商品关联，用于自动发货

### 5. 自动回复测试

1. 点击左侧菜单【自动回复】→【关键词回复】
2. 查看预置的4条自动回复规则
3. 测试添加规则：设置触发关键词、回复内容、匹配方式
4. 测试规则优先级：数值越大优先级越高

### 6. 指定回复测试

1. 点击左侧菜单【自动回复】→【指定回复】
2. 查看预置的2条指定回复（针对特定用户/商品）
3. 添加针对特定买家的专属回复

### 7. 在线聊天测试

1. 点击左侧菜单【在线聊天】
2. 查看预置的3个聊天会话
3. 点击会话查看历史消息
4. 测试发送消息功能

### 8. 系统设置测试

1. 点击左侧菜单【系统设置】
2. 测试修改系统配置（自动回复开关、库存预警阈值等）

## 题目内容
帮我开发一套闲鱼管理系统，PHP语言，数据库MYSQL，规范代码，写清注释，UI好看，自适应电脑手机平板，并且具备API接口方便易语言接入，具体功能如下，账号管理，商品管理，订单管理，自动回复，指定回复，卡券管理，自动发货，消息提醒，在线聊天。

### 功能需求

本系统需实现以下核心功能模块：

1. **账号管理** - 管理多个闲鱼账号，支持分组、状态监控
2. **商品管理** - 商品信息维护、上下架、分类管理
3. **订单管理** - 订单查询、发货、状态跟踪
4. **自动回复** - 基于关键词的自动回复规则配置
5. **指定回复** - 针对特定用户/商品的定制回复
6. **卡券管理** - 虚拟商品卡密管理、批量导入
7. **自动发货** - 订单支付后自动发送卡密
8. **消息提醒** - 新订单、库存预警等通知
9. **在线聊天** - 与买家的实时聊天记录管理

### 技术要求

- **后端**: PHP 8.2 + Apache, MySQL 8.0, RESTful API
- **前端**: React 18, Ant Design 5, Vite, Nginx
- **部署**: Docker, Docker Compose
- **架构**: 同时支持 ARM64 和 AMD64（x86_64）

> 注意：MySQL 8.0 官方镜像原生支持多架构，无需额外配置

### API 接口文档

系统提供完整的 RESTful API 接口，支持第三方（如易语言）接入。

**Swagger 在线文档**：[http://localhost:8084/docs/](http://localhost:8084/docs/)

文档包含：
- 全部 API 接口定义（认证、账号、商品、订单、卡券、回复、聊天、通知、配置）
- 请求/响应参数说明
- 在线接口调试功能
- JWT Token 和 API Key 两种认证方式说明

> API Key 可在【系统设置】→【API接口】中查看和刷新

## 常见问题

**Q: 登录后立即跳转回登录页？**
A: 请清除浏览器缓存后重试，或使用无痕模式访问。

**Q: 数据库连接失败？**
A: 首次启动时 MySQL 初始化需要时间，请等待约30秒后重试。

**Q: 如何重置测试数据？**
A: 执行 `docker-compose down -v && docker-compose up --build -d` 重新初始化。
